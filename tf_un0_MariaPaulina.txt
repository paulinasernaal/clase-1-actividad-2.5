1.Un sistema de control de versiones es una herramienta que permite gestionar y registrar los cambios realizados en archivos a lo largo del tiempo. Facilita la colaboración entre varios desarrolladores, asegurando que cada versión del archivo sea accesible y reversible. Además, permite comparar, restaurar y fusionar diferentes versiones de los archivos.
2.Un repositorio local es un almacén de código que reside en el disco duro de tu máquina, donde puedes hacer cambios y commits. Un repositorio remoto es una versión del repositorio alojada en un servidor, accesible a través de la red, permitiendo colaboración y sincronización con otros desarrolladores. La comunicación entre ambos se realiza mediante comandos de Git como push y pull.
3. Git es un sistema de control de versiones que gestiona y rastrea los cambios en el código fuente. GitHub es una plataforma de alojamiento de repositorios Git en la nube que facilita la colaboración, integración continua y la gestión de proyectos. Git es una herramienta de línea de comandos, mientras que GitHub ofrece una interfaz web y servicios adicionales.
4.Area: carpeta que contiene los archivos y el directorio.git del repositorio. Area de preparación: contiene el conjunto de archivos y cambios que se incluiran en el proximo commit. Directorio git: contiene los metadatos y las versiones de proyecto
5.Las ramas en Git se utilizan para desarrollar funcionalidades, corregir errores o experimentar de forma aislada sin afectar la rama principal (generalmente main o master). Permiten a los desarrolladores trabajar en paralelo, facilitando la integración de cambios cuando están listos. Las ramas también ayudan a gestionar el flujo de trabajo y a mantener el historial de cambios organizado.
